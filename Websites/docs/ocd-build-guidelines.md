# OCD WordPress Build Guidelines

System prompt for Claude Code when building OCD client sites. Copy this file into each client project root as `CLAUDE.md` so the agent loads it automatically. Output of a build is a complete custom theme folder ready for an install agent to drop into `/wp-content/themes/`.

---

## 1. Purpose and scope

You are building a production WordPress theme for an OCD client. The deliverable is a self-contained theme folder. An install agent handles activation, page creation, plugin install, and Gravity Forms import. You do not build dashboards, plugins, or anything outside the theme.

In scope:
- Custom theme (`style.css`, `functions.php`, templates, template parts)
- ACF Pro field groups synced via local JSON
- Gravity Forms embedded via template code (not shortcodes pasted into content)
- All PHP, CSS, JS required for the front-end

Out of scope:
- Plugin development
- Multisite configuration
- Server provisioning
- DNS, SSL, email setup
- GTM and tag installation (handled separately by Mahesh via HFCM)

---

## 2. Stack

Required:
- WordPress current stable
- PHP 8.1+
- ACF Pro (with local JSON sync enabled)
- Gravity Forms
- Vanilla CSS using custom properties
- Vanilla JS (jQuery only if a third-party library demands it)

Forbidden:
- Page builders: Elementor, Bricks, Divi, Beaver, Oxygen, Breakdance
- Other form plugins: WPForms, CF7, Fluent Forms, Ninja Forms
- CSS frameworks: Tailwind, Bootstrap, Bulma, Foundation
- JS frameworks: React, Vue, Svelte, Alpine
- Preprocessors: SCSS, LESS, PostCSS
- Build tooling: npm, webpack, vite, gulp
- Composer dependencies inside the theme
- Inline styles or inline scripts beyond critical preload tags
- Removing, disabling, or filtering out default WordPress functions, actions, hooks, settings, or behaviour (see rule below)

### Hard rule: never remove WordPress defaults

Default WordPress functionality stays intact in every build. The theme adds capability, it never strips it.

This means no:
- `remove_action()` or `remove_filter()` calls against core actions and filters
- Disabling emoji scripts, oembed discovery, RSS feeds, REST API, XML-RPC, or wp-cron
- Disabling comments, pingbacks, or trackbacks (leave them off via Settings if the client wants them off; the install agent toggles those in the admin, not the theme code)
- Removing default image sizes
- Hiding default admin menu items, dashboard widgets, or the admin bar
- Filtering out core scripts or styles enqueued by WordPress

If a default behaviour creates a real problem on a specific build, stop and ask before adding any removal code.

If a requirement genuinely can't be met within these constraints, stop and ask.

## 3. Theme architecture

One custom theme per client. No child theme. No shared base theme. Each build is fully self-contained.

```
clientslug/
├── style.css                       (theme header only, no actual styles)
├── functions.php                   (bootstrap, requires inc/ files)
├── header.php
├── footer.php
├── index.php                       (fallback, rarely hit)
├── page.php                        (default page template)
├── 404.php
├── searchform.php
├── screenshot.png                  (1200x900, branded)
├── README.md                       (install handover doc, see section 12)
├── page-templates/
│   ├── landing-page.php
│   ├── services.php
│   ├── contact.php
│   └── thank-you.php
├── template-parts/
│   ├── hero.php
│   ├── services-grid.php
│   ├── testimonials.php
│   ├── faq.php
│   ├── cta-band.php
│   ├── contact-block.php
│   └── service-areas.php
├── inc/
│   ├── theme-setup.php
│   ├── enqueue.php
│   ├── acf-config.php
│   ├── acf-options.php
│   ├── gravity-forms.php
│   └── helpers.php
├── acf-json/
│   └── group_xxx.json              (auto-generated by ACF on save)
└── assets/
    ├── css/
    │   └── main.css                (single stylesheet)
    ├── js/
    │   └── main.js                 (single script)
    ├── fonts/                      (self-hosted, woff2 only)
    └── images/                     (theme-level static assets only)
```

Client uploads go through the Media Library, not `assets/images/`.

## 5. functions.php

Minimal bootstrap. All actual logic in `inc/` files.

```php
<?php
if (!defined('ABSPATH')) { exit; }

define('OCD_THEME_VERSION', '1.0.0');
define('OCD_THEME_PATH', get_template_directory());
define('OCD_THEME_URI', get_template_directory_uri());

require_once OCD_THEME_PATH . '/inc/theme-setup.php';
require_once OCD_THEME_PATH . '/inc/enqueue.php';
require_once OCD_THEME_PATH . '/inc/acf-config.php';
require_once OCD_THEME_PATH . '/inc/acf-options.php';
require_once OCD_THEME_PATH . '/inc/gravity-forms.php';
require_once OCD_THEME_PATH . '/inc/helpers.php';
```

Bump `OCD_THEME_VERSION` on every CSS or JS change so cache-busting works.

## 6. Theme setup (inc/theme-setup.php)

```php
<?php
if (!defined('ABSPATH')) { exit; }

add_action('after_setup_theme', function() {
    add_theme_support('title-tag');
    add_theme_support('post-thumbnails');
    add_theme_support('html5', ['search-form', 'gallery', 'caption', 'style', 'script']);
    add_theme_support('responsive-embeds');

    register_nav_menus([
        'primary' => 'Primary Navigation',
        'footer'  => 'Footer Navigation',
    ]);

    add_image_size('hero', 1920, 1080, true);
    add_image_size('card', 600, 400, true);
    add_image_size('thumb-sq', 400, 400, true);
});
```

Do not remove default WordPress actions, filters, or behaviour from this file. No stripping emoji scripts, no removing oembed, no disabling comments, no `remove_action` or `remove_filter` calls against core. See section 2 for the full rule.

## 7. Enqueue (inc/enqueue.php)

```php
<?php
if (!defined('ABSPATH')) { exit; }

add_action('wp_enqueue_scripts', function() {
    wp_enqueue_style(
        'ocd-main',
        OCD_THEME_URI . '/assets/css/main.css',
        [],
        OCD_THEME_VERSION
    );

    wp_enqueue_script(
        'ocd-main',
        OCD_THEME_URI . '/assets/js/main.js',
        [],
        OCD_THEME_VERSION,
        ['in_footer' => true, 'strategy' => 'defer']
    );
});
```

Single CSS file, single JS file, both versioned. Defer the script.

## 8. ACF Pro conventions

### 8.1 Local JSON sync (inc/acf-config.php)

```php
<?php
if (!defined('ABSPATH')) { exit; }

add_filter('acf/settings/save_json', function() {
    return OCD_THEME_PATH . '/acf-json';
});

add_filter('acf/settings/load_json', function($paths) {
    unset($paths[0]);
    $paths[] = OCD_THEME_PATH . '/acf-json';
    return $paths;
});
```

Field groups created in WP admin save to `/acf-json/` automatically. The install agent gets field groups on theme activation without needing to import anything manually.

### 8.2 Options page (inc/acf-options.php)

Site-wide content (phone, email, address, social links, footer text, schema info) lives on an Options page, not duplicated per page.

```php
<?php
if (!defined('ABSPATH')) { exit; }

add_action('acf/init', function() {
    if (function_exists('acf_add_options_page')) {
        acf_add_options_page([
            'page_title' => 'Site Options',
            'menu_title' => 'Site Options',
            'menu_slug'  => 'site-options',
            'capability' => 'edit_posts',
            'redirect'   => false,
            'icon_url'   => 'dashicons-admin-generic',
            'position'   => 2,
        ]);
    }
});
```

Standard Options page fields every client gets:
- Business phone (tel format)
- Display phone (formatted for humans)
- Business email
- Business address (full)
- Suburb, state, postcode (for schema)
- Trading hours (repeater: day, open, close, closed boolean)
- Social links (repeater: platform, URL)
- Footer disclaimer / copyright line
- Schema business type (text)
- Schema ABN (text, optional)

### 8.3 Naming

- Field group title: human readable, e.g. `Hero Section`
- Field group key: `group_[section]_[template]`, e.g. `group_hero_landing`
- Field name (slug): `snake_case`, e.g. `hero_headline`, `cta_button`
- Field key: `field_[unique]`, must be unique across the site

### 8.4 Field group rules

- One field group per page template, attached via location rule `Page Template = [template name]`
- Hide the default editor and other unneeded panels via Hide on Screen: editor, comments, discussion, custom fields, slug, author, format, page attributes (unless needed)
- Repeaters for any list-like content (services, testimonials, FAQ items, team members, service areas)
- Use Group fields to bundle related fields (e.g. CTA group: label, URL, style)

### 8.5 Output in templates

Always escape on output. Always.

```php
echo esc_html(get_field('hero_headline'));
echo wp_kses_post(get_field('about_copy'));     // WYSIWYG only
echo esc_url(get_field('cta_url'));
echo esc_attr(get_field('aria_label'));
```

Never echo raw `get_field()`. No exceptions.

### 8.6 Image fields

Set Return Format to `Image Array`. Output via `wp_get_attachment_image()` so you get native srcset and lazy loading:

```php
$img = get_field('hero_image');
if ($img) {
    echo wp_get_attachment_image(
        $img['ID'],
        'hero',
        false,
        [
            'class' => 'hero__image',
            'alt'   => esc_attr($img['alt'] ?: ''),
        ]
    );
}
```

### 8.7 Link fields

Use the Link field type. It returns `['title' => ..., 'url' => ..., 'target' => ...]`. Helper in `inc/helpers.php`:

```php
function ocd_render_link($link, $class = 'btn btn--primary') {
    if (!$link || empty($link['url'])) { return; }
    $target = !empty($link['target']) ? ' target="_blank" rel="noopener"' : '';
    printf(
        '<a href="%s" class="%s"%s>%s</a>',
        esc_url($link['url']),
        esc_attr($class),
        $target,
        esc_html($link['title'])
    );
}
```

## 9. Gravity Forms conventions

### 9.1 Embedding

Embed in PHP templates, never via shortcode pasted into a content field.

```php
<div class="ocd-form ocd-form--enquiry">
    <?php
    if (function_exists('gravity_form')) {
        gravity_form(1, false, false, false, null, true);
    }
    ?>
</div>
```

The form ID (`1` above) should come from an ACF number field on the page so the client can swap forms without touching code.

```php
$form_id = get_field('form_id');
if ($form_id && function_exists('gravity_form')) {
    gravity_form($form_id, false, false, false, null, true);
}
```

### 9.2 Mandatory form configuration

Document in README.md for the install agent. These settings are non-negotiable on every form:

- Notification BCC: `hello@onlinecreativedudes.com` (always)
- Reply-To: mapped to the form's email field
- Phone fields: International format (not US/Standard)
- Confirmation: redirect to `/thank-you/` on this site
- AJAX submission: enabled
- Honeypot: enabled
- Form title and description: hidden in form settings (we render headlines in template code)

### 9.3 Styling

Wrap every form in `.ocd-form`. Scope all GF overrides under that class. Do not edit GF's admin stylesheets. The install agent does not need to touch CSS.

```css
.ocd-form .gform_wrapper .gfield label { /* ... */ }
.ocd-form .gform_wrapper input[type="text"] { /* ... */ }
.ocd-form .gform_wrapper .gform_button { /* ... */ }
```

### 9.4 Thank You page (mandatory)

Every site has a Thank You page. No exceptions. It is the confirmation hook for every Gravity Form on the site.

Build requirements:
- File: `page-templates/thank-you.php` with `Template Name: Thank You`
- Page slug at install: `/thank-you/`
- Page title: `Thank You`
- Page assigned the Thank You template
- Index excluded: include `<meta name="robots" content="noindex, nofollow">` output in `header.php` when this template is active, so the page doesn't get indexed by Google
- Conversion tracking placeholder div present in the template so Mahesh can wire GTM triggers later: `<div id="conversion-event" data-event="form_submission"></div>`

Form binding requirement: every Gravity Form on the site, without exception, has its confirmation set to **Page** redirect, target `/thank-you/`. Not a confirmation message. Not a different page. Always `/thank-you/`. The install agent verifies this on every form before sign-off.

The template structure:

```php
<?php
/**
 * Template Name: Thank You
 */
if (!defined('ABSPATH')) { exit; }

get_header();

$headline = get_field('ty_headline') ?: 'Thanks, we got your message.';
$body     = get_field('ty_body');
$cta      = get_field('ty_cta');
?>

<main class="page page--thank-you">
    <section class="thank-you">
        <div class="thank-you__container container container--narrow">
            <h1 class="thank-you__headline"><?php echo esc_html($headline); ?></h1>

            <?php if ($body) : ?>
                <div class="thank-you__body">
                    <?php echo wp_kses_post($body); ?>
                </div>
            <?php endif; ?>

            <?php ocd_render_link($cta, 'btn btn--primary thank-you__cta'); ?>

            <div id="conversion-event" data-event="form_submission" aria-hidden="true"></div>
        </div>
    </section>
</main>

<?php get_footer(); ?>
```

ACF field group for this template (`group_thankyou_page`):
- `ty_headline` (text) - defaults handled in template if empty
- `ty_body` (WYSIWYG) - optional
- `ty_cta` (link) - typically points back to homepage

README.md must include the Thank You page in the page creation checklist as the first page to create, with an explicit note that all form confirmations must point at it.

## 10. CSS conventions

### 10.1 Single file structure

All CSS in `assets/css/main.css`. Organised in this order:

```
/* 1. Tokens (CSS custom properties) */
/* 2. Reset / base */
/* 3. Typography */
/* 4. Layout primitives (.container, .stack, .grid) */
/* 5. Components (buttons, cards, forms, nav) */
/* 6. Sections (hero, services, testimonials, faq, cta-band, contact) */
/* 7. Page-specific overrides */
/* 8. Utilities */
/* 9. Media queries scoped per component or at end */
```

### 10.2 Tokens

All brand values as custom properties on `:root`. No raw hex codes outside `:root`. Fill these in per client at the top of `main.css`:

```css
:root {
    /* Colour */
    --c-bg: #ffffff;
    --c-surface: #f7f7f7;
    --c-text: #0d0d0d;
    --c-text-muted: #6b6b6b;
    --c-accent: #29c4c4;
    --c-accent-dark: #1ea4a4;
    --c-border: #e5e5e5;
    --c-error: #c53030;
    --c-success: #2f855a;

    /* Typography */
    --ff-display: 'Barlow Condensed', sans-serif;
    --ff-body: 'Barlow', sans-serif;

    --fs-base: 1rem;
    --fs-sm: 0.875rem;
    --fs-lg: 1.125rem;
    --fs-h1: clamp(2rem, 5vw, 3.5rem);
    --fs-h2: clamp(1.5rem, 3.5vw, 2.5rem);
    --fs-h3: clamp(1.25rem, 2.5vw, 1.75rem);

    --lh-tight: 1.1;
    --lh-base: 1.5;

    /* Spacing */
    --space-2xs: 0.25rem;
    --space-xs: 0.5rem;
    --space-sm: 1rem;
    --space-md: 2rem;
    --space-lg: 4rem;
    --space-xl: 6rem;
    --space-2xl: 8rem;

    /* Layout */
    --container: 1200px;
    --container-narrow: 800px;
    --radius: 6px;
    --radius-lg: 12px;

    /* Motion */
    --ease: cubic-bezier(0.4, 0, 0.2, 1);
    --transition: 200ms var(--ease);
}
```

### 10.3 Naming

BEM. `.block`, `.block__element`, `.block--modifier`. No utility soup, no ID selectors for styling, no nested compound selectors more than 2 deep. No `!important` except for one-off overrides of vendor CSS, always with an inline comment explaining why.

### 10.4 Responsive

Mobile first. Default styles target mobile. Layer up with `min-width` media queries. Breakpoints:

```css
/* sm */ @media (min-width: 640px) { }
/* md */ @media (min-width: 900px) { }
/* lg */ @media (min-width: 1200px) { }
```

Co-locate media queries with the component they affect, not in a separate section at the bottom.

### 10.5 Reset

Minimal reset at the top of `main.css`. No `normalize.css` import. Box-sizing, margin reset, image defaults, that's it.

```css
*, *::before, *::after { box-sizing: border-box; }
body { margin: 0; }
img, picture, video, svg { max-width: 100%; display: block; }
button, input, select, textarea { font: inherit; }
```

## 11. JS conventions

### 11.1 Single file

All JS in `assets/js/main.js`. Vanilla. No dependencies.

### 11.2 Pattern

```javascript
(function() {
    'use strict';

    const initMobileNav = () => {
        const toggle = document.querySelector('.nav__toggle');
        const menu = document.querySelector('.nav__menu');
        if (!toggle || !menu) return;
        toggle.addEventListener('click', () => {
            const open = menu.classList.toggle('is-open');
            toggle.setAttribute('aria-expanded', open);
        });
    };

    const initFAQ = () => {
        document.querySelectorAll('.faq__item').forEach(item => {
            const trigger = item.querySelector('.faq__question');
            if (!trigger) return;
            trigger.addEventListener('click', () => {
                item.classList.toggle('is-open');
            });
        });
    };

    const initStickyHeader = () => { /* ... */ };
    const initSmoothScroll = () => { /* ... */ };

    document.addEventListener('DOMContentLoaded', () => {
        initMobileNav();
        initFAQ();
        initStickyHeader();
        initSmoothScroll();
    });
})();
```

### 11.3 JS scope

JS handles:
- Mobile nav toggle
- Accordion and FAQ open/close
- Smooth scroll for anchor links
- Sticky header on scroll
- Simple show/hide interactions

JS does not handle:
- Animations beyond class toggling (use CSS transitions/animations)
- Content rendering (server renders, ACF outputs, JS enhances)
- Form validation (Gravity Forms handles)
- Sliders or carousels unless explicitly required by the brief (and then only via a tiny vanilla library like Swiper or Splide, self-hosted)

## 12. PHP and WordPress conventions

### 12.1 Standards

- WordPress PHP Coding Standards
- `snake_case` for functions and variables
- `PascalCase` for classes (rare in theme code)
- Prefix every custom function with `ocd_`

### 12.2 Security

Every PHP file starts with:

```php
<?php
if (!defined('ABSPATH')) { exit; }
```

Escape every output:
- `esc_html()` for text
- `esc_attr()` for attribute values
- `esc_url()` for URLs
- `wp_kses_post()` for WYSIWYG HTML

Sanitise every input:
- `sanitize_text_field()` for plain text
- `sanitize_email()` for email
- `absint()` for integers
- `wp_kses_post()` for HTML inputs

Nonces on any custom form handling. No raw `$_POST` or `$_GET` access.

### 12.3 No logic in templates

Templates render. Helpers in `inc/helpers.php` compute. If a template has more than three lines of logic before the first HTML tag, extract a helper.

## 14. Template part pattern

```php
<?php
/**
 * Hero section
 * ACF group: group_hero_landing
 */
if (!defined('ABSPATH')) { exit; }

$headline = get_field('hero_headline');
$subhead  = get_field('hero_subhead');
$cta      = get_field('hero_cta');
$image    = get_field('hero_image');

if (!$headline) { return; }
?>

<section class="hero">
    <div class="hero__container container">
        <div class="hero__content">
            <h1 class="hero__headline"><?php echo esc_html($headline); ?></h1>

            <?php if ($subhead) : ?>
                <p class="hero__subhead"><?php echo esc_html($subhead); ?></p>
            <?php endif; ?>

            <?php ocd_render_link($cta, 'btn btn--primary hero__cta'); ?>
        </div>

        <?php if ($image) : ?>
            <div class="hero__media">
                <?php echo wp_get_attachment_image($image['ID'], 'hero', false, [
                    'class' => 'hero__image',
                    'alt'   => esc_attr($image['alt'] ?: ''),
                ]); ?>
            </div>
        <?php endif; ?>
    </div>
</section>
```

A template part that has no content (required field missing) returns silently. Never render an empty `<section>`.

## 15. Performance

- All images via `wp_get_attachment_image()` for native srcset and `loading="lazy"`
- Hero image gets `<link rel="preload">` in `header.php` for the landing template
- Self-host fonts in `assets/fonts/` as woff2 with `font-display: swap`
- Single CSS file, single JS file
- JS deferred via the enqueue strategy
- No external font CDN unless explicitly required
- No third-party scripts in the theme (GTM and tags go in via HFCM later)

## 16. SEO basics

- `title-tag` theme support handles `<title>` (no manual tag)
- Meta description: ACF field on every page, output in `header.php`
- Open Graph and Twitter Card tags in `header.php` pulling from page ACF + featured image
- Canonical link in `header.php`
- Schema.org LocalBusiness JSON-LD in `footer.php` pulling from Options page
- Breadcrumb schema where relevant
- `robots.txt` and `sitemap.xml` handled by Yoast or Rank Math at install time (note in README)

Example schema block in `footer.php`:

```php
<?php
$phone = get_field('business_phone', 'option');
$address = get_field('business_address', 'option');
$name = get_bloginfo('name');
?>
<script type="application/ld+json">
{
    "@context": "https://schema.org",
    "@type": "LocalBusiness",
    "name": "<?php echo esc_js($name); ?>",
    "telephone": "<?php echo esc_js($phone); ?>",
    "address": "<?php echo esc_js($address); ?>"
}
</script>
```

## 17. Accessibility baseline

- Semantic HTML (`<header>`, `<nav>`, `<main>`, `<section>`, `<footer>`, `<article>`)
- Single `<h1>` per page
- Skip-to-content link in `header.php`
- All interactive elements keyboard-reachable
- `aria-expanded` on mobile nav toggle and accordion triggers
- Visible focus styles (do not remove default outline without a clear replacement)
- All images have `alt` attributes (empty string for decorative, real alt for content)
- Form labels properly associated (Gravity Forms handles this if you don't break it)

## 18. README.md deliverable

Every theme folder ships with a `README.md` covering:

1. Theme name, client name, build date
2. Required plugins with version notes (ACF Pro, Gravity Forms, Yoast or Rank Math)
3. Install steps:
   - Upload theme folder to `/wp-content/themes/`

5. Options page setup:
   - List every field on Site Options and what to populate
6. Gravity Forms setup:
   - Number of forms to create
   - Form names and what page each lives on
   - Notification BCC reminder: `hello@onlinecreativedudes.com` (always)
   - Reply-To mapping reminder
   - Phone field format reminder: International
   - **Confirmation: every single form set to Page redirect, target `/thank-you/`. Verify on each form before sign-off.**
7. Menu setup: which menu items go where
8. Any third-party services to configure (Google Maps API key, etc.)
9. Known issues or notes for the install agent

## 20. Media & ACF image seeding (standard — applies to every build)

Bundled seed images must not look hard-coded. The client should open any page and
see real, editable images in the Media Library and in the ACF fields — not empty
fields with a file silently rendering behind them.

Every build ships an `inc/seed-media.php` and a `deploy/seed-media.php` runner that:

- **Import** every bundled image in `assets/images/` and `assets/brand/` into the
  WordPress Media Library, with a marker meta (`_pcp_seed_file` / `_ocd_seed_file`)
  so re-runs reuse the existing attachment instead of duplicating.
- **Populate** the matching ACF image fields — logo (options), homepage section
  images and repeater cards, and the hero/featured image on every CPT entry —
  **only where the field is currently empty**. Never overwrite a field that already
  has a value (so a client's real uploads are always safe).
- Are **idempotent and non-destructive**: safe to run on every deploy.

Template image helpers still keep a theme-file fallback for safety, but after the
media seed runs there should be no visible placeholder that isn't also a real,
swappable Media Library item wired to its ACF field. Run order at deploy:
content seed → services seed → **media seed**.
